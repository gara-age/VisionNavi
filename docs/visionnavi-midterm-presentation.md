# VisionNavi 중간발표 정리

## 1. 한 줄 소개

VisionNavi는 한국과 일본의 고령자를 위해, 음성이나 자연어 텍스트 명령만으로 웹 검색과 Windows 데스크톱 작업을 도와주는 로컬 AI 에이전트형 자동화 시스템입니다.  
단순한 매크로나 고정된 스크립트가 아니라, 사용자의 요청을 이해하고 실제 화면 상황을 반영해 작업을 끝까지 수행하는 방향을 목표로 하고 있습니다.

## 2. 왜 필요한가

고령자에게 웹과 PC 사용은 여전히 어렵습니다.

- 필요한 정보를 찾기 위해 검색어를 직접 잘 입력해야 합니다.
- 사이트 구조가 복잡해 원하는 버튼이나 메뉴를 찾기 어렵습니다.
- 메모 작성이나 파일 확인 같은 간단한 작업도 여러 단계를 거쳐야 합니다.

기존 자동화도 있었지만 한계가 분명했습니다.

- 처음에 정한 step이 화면 변화에 약했습니다.
- 중간에 예외가 생기면 복구가 어려웠습니다.
- 사용자의 목표보다 정해진 순서를 재생하는 데 가까웠습니다.

그래서 VisionNavi는 "상황을 보며 목표를 끝까지 수행하는 자동화"를 지향합니다.

## 3. VisionNavi는 어떻게 해결하는가

VisionNavi의 기본 흐름은 단순합니다.

1. 사용자가 말하거나 입력합니다.
2. 시스템이 명령을 이해해 어떤 작업인지 정리합니다.
3. 웹 작업인지 데스크톱 작업인지 판단합니다.
4. 적절한 실행 채널을 선택합니다.
5. 실제 브라우저나 Windows 앱을 조작합니다.
6. 결과를 확인하고 사용자에게 안내합니다.

핵심은 VisionNavi가 단순 executor가 아니라 `orchestrator`라는 점입니다.  
즉, 입력 해석, 실행 채널 선택, 결과 검증, 실패 시 fallback까지 하나의 흐름으로 관리합니다.

## 4. 현재 어디까지 구현되었는가

현재 VisionNavi는 다음 기능을 실제로 구현해 두었습니다.

- 검색 및 읽기
  - 검색 엔진 또는 사이트에서 정보를 찾고 요약하는 흐름
- 메모장 작성
  - 메모장을 열고 내용을 입력하고 저장하는 자동화
- 음성 입력
  - 음성 파일 기반 STT와 실시간 음성 입력 흐름
- wakeword 실험
  - 호출어 감지 후 음성 입력을 시작하는 구조
- 고령자 친화 UI 개편
  - 큰 말하기 버튼, 생활형 문구, 단순한 메인 화면

다만 아직 완성 단계는 아닙니다.

- `search_and_read`는 비교적 앞서 있는 시나리오입니다.
- `open_notepad_and_type`는 external desktop agent를 검증하는 대표 시나리오입니다.
- `find_map_route`는 아직 internal baseline 비중이 큰 중간 단계입니다.

즉, 일부 시나리오는 agent형으로 발전하고 있지만, 모든 작업이 동일한 수준으로 완성된 것은 아닙니다.

## 5. 차별점은 무엇인가

VisionNavi의 차별점은 4가지로 정리할 수 있습니다.

### 1. 단순 매크로가 아니라 orchestrator 구조

명령 해석, 실행기 선택, 검증, fallback을 하나의 흐름으로 묶었습니다.

### 2. local AI stack

로컬 STT, wakeword, LLM, VLM을 결합해 외부 의존도를 줄이고 실사용형 구조를 만들고 있습니다.

### 3. external-first 방향

내부 실행기를 유지하되, 장기적으로는 BrowserUse / ComputerUse류 오픈소스 agent runtime을 통합해 범용성을 넓히는 방향입니다.

### 4. 고령자용 UX를 함께 설계

기술 데모가 아니라 실제 사용자를 고려해, 큰 조작 영역과 단순한 문구 중심의 UI를 동시에 개선하고 있습니다.

## 6. 시연에서 보여줄 내용

이번 시연에서는 다음 흐름을 보여줄 수 있습니다.

- 음성 또는 텍스트로 요청 입력
- 검색/읽기 자동화
- 메모장 작성 자동화
- 결과 팝업과 진행 안내

즉, VisionNavi가 단순히 명령을 해석하는 데서 끝나는 것이 아니라, 실제 브라우저와 데스크톱에서 작업을 수행하는 모습을 보여주는 것이 핵심입니다.

## 7. 현재 한계와 다음 단계

현재 가장 중요한 과제는 단순한 모델 교체보다, 실행 구조를 더 안전하고 일관되게 만드는 것입니다.

우선 과제는 다음과 같습니다.

- provider / query / language constraint 고정 강화
- external browser 실행 안정화
- external desktop 실행 안정화
- 일본어 wakeword / STT 품질 개선

즉 앞으로는 "LLM이 아무렇게나 바꾸지 못하게 하는 구조"와 "external agent를 더 안정적으로 쓰는 구조"를 우선 강화할 계획입니다.

## 8. 마무리

VisionNavi는 고령자가 더 쉽게 웹과 PC를 사용할 수 있도록, 음성 기반의 실질적인 자동화 도우미를 만드는 프로젝트입니다.  
현재는 검색, 메모장 작성, 음성 입력, wakeword, 고령자 친화 UI까지 중간 단계 구현을 마친 상태입니다.  
앞으로는 실행 안정성과 agent 통합 완성도를 높여, 실제 생활 속에서 쓸 수 있는 자동화 시스템으로 발전시키려 합니다.
